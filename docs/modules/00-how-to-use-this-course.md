# Module 0: How To Use This Course

Source tag: `Stable concept`

## Purpose

This is not a static book. It is a working course repository plus an
interactive tutoring loop.

You will learn best if you treat each module as four different activities:

1. Concept learning
2. Source reading
3. Exercise work
4. Interactive review with Codex

## Why this course is repo-first

Effect v4 is currently in beta, and the `effect-smol` repository provides the
strongest available evidence for idiomatic v4 usage.

Use public docs for stable concepts.
Use `effect-smol` for v4 API shape and teaching examples.
Do not assume that an older article or snippet matches beta v4 unless it
matches the repo.

## The learning loop

For each module:

1. Read the lesson notes.
2. Read the linked `effect-smol` source files.
3. Solve the exercise.
4. Ask Codex to review your understanding before moving on.

## What to ask Codex during the course

- "Quiz me on Module N."
- "Review my exercise solution."
- "Explain why this service needs `R`."
- "Compare this to Promise code."
- "Point out every mistake in my mental model."

## What not to do

- Do not try to memorize the entire API surface.
- Do not start with unstable subsystems just because they look exciting.
- Do not treat `Effect` as a fancy Promise and ignore `R` and resource safety.
- Do not trust unsourced explanations when the repo says something else.

## Completion criteria for each module

You should be able to:

- explain the concept in your own words
- map it to at least one repo example
- implement one small exercise
- compare it to a non-Effect approach
